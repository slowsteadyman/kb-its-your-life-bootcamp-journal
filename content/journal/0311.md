## javascript

- dom (document object model)
  - 자바스크립트는 모든 html 요소를 갖고 부모 자식 관계에 따라 dom tree를 형성함
  - dom tree 객체를 document로 접근 가능함
- 자바스크립트 전역객체 window
- querySelector가 반환하는 것은 유사 배열 객체임
  = result[1], result.length 모두 가능하지만 배열은 아님
  - 인덱스처럼 보이는 것은 사실 key임
  - foreach 같은 배열 메소드 사용 불가능 함
- **var, let, const의 차이점**
  - var는 블럭 안에서 재선언하면 블럭을 나와도 그 값이 유지된다는 문제점이 있음
    - if, for, while과 같은 블럭은 완전 무시함
    - 함수 내부에서는 지역 변수, 함수 바깥에서는 전역 변수
  - let은 한번 선언하고 재할당이 가능하고
  - const는 상수니까 딱 한번만 선언할 수 있음
- js로 css 수정할 때는 div.style.background-color 이런 거 안 됨
  - '-'를 빼기로 인식함
- 사용자 속성이라는 것이 있음
  - 'data-'라는 접두사를 붙임으로써 지정할 수 있음
  - 객체 속성 이름에는 'data-' 뒷부분만 들어감
  - 속성 조작 메소드를 통해 값을 수정할 수 있음
- setInterval(event handler, time)
  - 일정 간격마다 함수를 실행
- event handler에서 this는 그 핸들러를 호출한 객체가 this가 됨
  - setInterval 의 경우에는 window가 됨
- a tag의 href 연결은 디폴트로 등록된 이벤트 핸들러임
  - 핸들러 함수를 직접 정의함으로써 제지할 수 있음
- **querySelectorAll을 사용하면 조건에 맞는 객체가 하나여도 nodeList 안에 담겨져서 반환됨**
  - 그러므로 querySelectorAll()[0]을 써주거나
  - getElementById()를 써주면 됨
- callback 함수
  - 나중에 준비되면 다시 전화해줘
  - 언제 끝날지 모르는 작업에 대해 그 작업이 끝나고 바로 무언가를 시키고 싶을 때 사용함
- 람다식 (() => {})을 사용하는 이유
  - this 때문임
  - this.name 이런식으로 객체의 필드 값을 사용할 때가 있음
  - 근데 어떤 콜백 함수 안에서 this를 사용할 경우 this는 콜백함수를 호출한 window가 될 수 있음
  - 꼭 window인 것은 아니지만 this가 바로 상위 객체를 가리키지 않을 위험성이 있다는 것
  - 그래서 일반 함수를 통해 this를 사용하는게 아니라 람다식을 통해 this를 호출함
  - 그렇게 하면 화살표 함수는 자신만의 this를 갖지 않고 상위의 this를 그대로 물려받음

## 배열 메소드

- Array.from(object, function())
  - 객체의 값을 변형시키면서 배열을 생성할 수 있음
- reduce
  - 배열의 값을 하나로 합치는 메소드
  - ex) 초기값 10에 모든 배열 값을 더하기
- 메소드 인자로 함수를 줘서 메소드 작업을 더 디테일하게 수행할 수 있음
  - array.sort(funtion() compare)
