- DNS (Domain Name System)
- http 프로토콜은 `1. 절차 2. 메시지 내용`으로 구성된다
- TCP (Transmission Control Protocol) IP (Internet Protocol)
  - tcp는 양 끝단에서 사용되는 end to end 프로토콜
  - ip는 중간의 라우터들 간의 통신에 사용되는 프로토콜
  - 애플리케이션은 고유 포트 번호를 통해 메시지를 수신할 수 있다
    - 우편물이 우리집에 왔는데, 우리집의 누구한테 왔는지가 있어야지
- HTTP (HyperText Transfer Protocol)
  - http 프로토콜이라고 하면 안되는거네
    - htt 프로토콜이라고 하거나 http 라고 해야되네
  - http는 stateless임
    - 이전 통신 내용을 기억하지 않음
- http request 양식
  - [request line] httpMethod URL protcol/Version
  - [request header] fieldname: content
  - [request body] get 요청 시 body 없음
  - 헤더 바디는 \n으로 구분
- http response도 비슷한 구성임
- response body는 서버가 조작/수정할 수 있음 -> ==백엔드 개발==
- url에서 80번(디폴트) 포트가 아니면 반드시 기재해줘야 함
- semantic 태그를 통해 css를 더 효율적으로 쓸 수 있음
