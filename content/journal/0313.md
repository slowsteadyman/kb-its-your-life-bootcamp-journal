## 동기, 비동기

- # 프로세스와 스레드의 차이점
  - 스레드에는 공유 자원이 있음
    - 멀티스레드 프로그래밍에서는 동기화 문제가 발생함
    - lock, synchronization에 대한 개발자의 책임이 있음
  - 싱글스레드는 느림
    - 성능 향상에 대한 트레이드오프가 있음
- 작업의 성격에 따라 자바스크립트는 멀티스레드보다 싱글스레드가 성능이 더 좋게 나옴
  - I/O 과정
    - cpu는 I/O 명령만 내리고, 컨트롤러가 실질적인 I/O를 처리함
    - cpu는 I/O를 만나면 프로세스를 중단시킴
    - I/O가 끝나면 인터럽트가 발생해 cpu가 다시 프로세스에 할당됨
    - -> 컴퓨터 시스템 자체는 비동기 시스템으로 운영됨
    - 하지만 사용자 관점에서 프로그램은 동기적으로 작동함
      - I/O에서 멈춰있음
      - 그래서 콜백을 통해 I/O 발생 시간동안 다른 작업을 할 수 있음
      - -> 마치 멀티스레드인 것처럼 작동함
  - I/O를 많이 수행하는 작업이 네트워크 서버임
    - 여기서 많이 사용함
- 자바스크립트에서 동기처리와 비동기처리
  - 콜스택이 모두 수행한 후 콜백큐의 남은 작업을 수행함
  - 그래서 setTimeout에 0초를 넣어줘도 마지막에 코드가 실행되는 것임
- try-catch로는 동기 함수의 에러만 잡아낼 수 있음
  - try-catch 안의 setTimeout의 에러는 잡아낼 수 있음
  - 하지만 setTimeout의 콜백함수의 에러는 잡아낼 수 없음
  - 따라서 setTImeout 안에 try-catch를 만들어야 함

- ## 스크립트 언어의 함수
  - 컴파일 언어(c)는 스택에 함수와 지역변수를 저장함
  - js, python은 스택에 함수 정보만 저장되고 변수는 힙에 할당됨
    - js에서 이 힙을 실행 context라고 함
    - 함수가 끝나도 스택만 없어지고 context는 없어지지 않음
    - garbage collector에 의해 따로 힙 메모리가 제거됨
    - 힙에 대한 다른 링크가 남아있으면 함수가 끝나도 변수가 살아있음
      - **이 영역을 closure(클로저) 라고 함**
